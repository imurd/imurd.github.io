# Реагирование: документирование и жалобы

Документ для **защитной** реакции, если тебя выцеливают. Никакого ответного взлома —
только сбор доказательств, атрибуция и обращение по официальным каналам.

---

## 1. Что уже собирает лендинг (доказательная база)

По каждому переходу в таблице `visits` (смотри `admin.html`, кнопка «Скачать CSV»):

- **Сетевое:** IP, ASN, провайдер (ISP), домен, флаг подозрения на VPN/proxy
  (`vpn_suspect` = пояс браузера ≠ пояс по IP).
- **Гео:** город, регион, страна.
- **Устройство:** ОС, браузер, тип, экран, окно, ядра CPU, ОЗУ, тип сети, языки, часовой пояс.
- **Контекст:** referrer (откуда пришёл), точное время, число визитов (сгруппировано по IP).

CSV из `admin.html` — готовое приложение к любой жалобе.

---

## 2. Атрибуция без атаки (как понять, куда жаловаться)

1. **Кому принадлежит IP** — определи хостера/провайдера и его abuse-контакт:
   - https://whois.arin.net/  или  `whois <IP>` в терминале;
   - по ASN (в таблице колонка ASN) — https://bgp.he.net/  → страница ASN → раздел Abuse/Contact.
2. **VPN/хостинг?** Если `vpn_suspect = ДА` или домен/провайдер похож на дата-центр —
   реальную личность по IP не установить, но abuse-жалоба хостеру всё равно работает
   (он видит своего клиента по логам).
3. **Referrer** показывает площадку (напр. `instagram.com`) — туда идёт отдельная жалоба.

> Определять личность/адрес по IP имеет право только оператор/провайдер по официальному
> запросу (полиция, суд). Твоя задача — собрать логи и передать их правильному адресату.

## 3. Шаблоны жалоб

### 3.1 Хостеру / провайдеру (abuse@)

```
To: abuse@<провайдер>
Subject: Abuse report — malicious/targeting activity from IP <IP> (AS<ASN>)

Hello,

I am reporting activity originating from an IP address in your network.

- IP address: <IP>
- ASN / network: AS<ASN> (<org>)
- Date/time (UTC): <дата и время из таблицы>
- Description: this IP was used to access a page set up to identify a targeting/
  phishing attempt against me. Full technical log (IP, timestamps, user-agent,
  geolocation) is attached as CSV.

Please investigate and take action according to your AUP. I can provide additional
logs on request.

Regards,
<Имя>
<контакт для ответа>
```

### 3.2 Площадка (Instagram / Meta)

- В приложении: профиль/сообщение → «…» → **Пожаловаться** → выбери «мошенничество/выдача себя за другого».
- Форма Meta: https://www.facebook.com/help/instagram/ (раздел Report Something).
- Приложи: ссылку на профиль/сообщение, скриншоты, время, и что тебя пытались выманить.

### 3.3 Полиция / CERT Молдовы

- **CERT-GOV-MD** (нацкоманда реагирования): https://cert.gov.md/  — форма/почта для инцидентов.
- **Полиция МД, киберпреступления:** https://politia.md/  → обращение; горячая линия 112.
- Приложи CSV из `admin.html`, переписку, описание, чего от тебя добивались.

> Совет: не удаляй исходные сообщения/ссылки — это доказательства. Сделай скриншоты
> с видимой датой.

## 4. Защита себя (чтобы перестать «светиться»)

**Не давай собирать свои данные так же, как это делает этот лендинг:**

- Не переходи по ссылкам из сообщений/соцсетей «на лету». Наводи курсор, смотри реальный
  домен; сокращённые ссылки (bit.ly, l.instagram.com редиректы) разворачивай через
  https://unshorten.it/ прежде чем открывать.
- Открывай подозрительные ссылки в **режиме инкогнито** или на изолированном устройстве —
  меньше cookie, отпечатка и истории.
- **VPN** скрывает реальный IP/город/провайдера (именно то, что этот лендинг ловит).
- В браузере: блокировщик трекеров (uBlock Origin), запрет сторонних cookie,
  анти-фингерпринт (Firefox с `resistFingerprinting`, или Brave).
- **Геолокацию** сайтам не разрешай (тот самый запрос «разрешить доступ к местоположению»).
- Отключи автозагрузку картинок в письмах (пиксель-трекеры выдают факт открытия).

**Гигиена аккаунтов:**

- Разные пароли + менеджер паролей; 2FA везде, где можно.
- Проверь утечки своих email/телефона: https://haveibeenpwned.com/
- Минимизируй публичные данные в соцсетях (город, место работы, телефон) — из них строят
  «портрет» для таргетинга.

**Если уже клюнул (ввёл данные / скачал файл):**

- Смени пароли с чистого устройства, отзови сессии, включи 2FA.
- Проверь устройство антивирусом; при сомнении — переустанови ОС.
- Предупреди банк, если могли утечь платёжные данные.
